# Ошибка JdkImageTransform / jlink — пути с кириллицей

Ошибка `Error while executing process ... jlink.exe` возникает, когда в путях есть **кириллица** (или другие не-латинские символы). Нужно, чтобы все пути были только **латиница** (и при необходимости цифры/подчёркивание).

---

## Что поменять

### 1. Папка Gradle (кэш) — убрать «Никита» из пути

Сейчас кэш лежит в `C:\Users\Никита\.gradle`. Перенаправь его в папку без кириллицы.

**Вариант A — через переменную окружения (рекомендуется):**

1. Нажми **Win + R**, введи `sysdm.cpl`, Enter.
2. Вкладка **Дополнительно** → **Переменные среды**.
3. В «Переменные среды пользователя» нажми **Создать**:
   - Имя: `GRADLE_USER_HOME`
   - Значение: `G:\.gradle` (или `C:\gradle` — любая папка **только с латиницей**).
4. OK везде. **Перезапусти Android Studio** (лучше и ПК перезагрузить).

**Вариант B — только для текущей сессии:**  
Перед запуском Android Studio открой **cmd** и выполни:
```bat
set GRADLE_USER_HOME=G:\.gradle
```
Потом из этого же окна запусти Android Studio (укажи полный путь к studio64.exe).

---

### 2. Путь к Android Studio (JBR) — убрать «андроид апп»

Сейчас: `G:\андроид апп\`. Оттуда вызывается `jlink.exe`, из-за кириллицы он падает.

**Что сделать:**

- **Либо** переустановить Android Studio, выбрав папку **только с латиницей**, например:
  - `G:\AndroidStudio`
  - или `C:\Program Files\Android\Android Studio`
- **Либо** скопировать всю папку установки (например, `G:\андроид апп`) в `G:\AndroidStudio`, затем запускать **studio64.exe** из `G:\AndroidStudio\bin\`. В настройках проектов ничего менять не нужно, важно, чтобы сам процесс студии и JBR были в пути без кириллицы.

---

### 3. Путь к Android SDK — убрать «андроид 2»

Сейчас: `G:\андроид 2\platforms\android-34\...`

**Что сделать:**

1. Скопируй папку с SDK (сейчас «андроид 2») в папку с латиницей, например: **G:\AndroidSDK** (или **G:\Android\sdk**).
2. В Android Studio: **File** → **Settings** (или **Ctrl+Alt+S**) → **Appearance & Behavior** → **System Settings** → **Android SDK**.
3. В поле **Android SDK Location** укажи новый путь, например: `G:\AndroidSDK`.
4. Нажми **Apply** / **OK**.

После этого сборка снова будет использовать SDK и jlink уже из путей без кириллицы.

---

## Порядок действий

1. Задать **GRADLE_USER_HOME** = `G:\.gradle` (или `C:\gradle`).
2. Перенести **Android Studio** в путь вида `G:\AndroidStudio` (или переустановить туда).
3. Перенести **Android SDK** в путь вида `G:\AndroidSDK` и обновить путь в настройках SDK.
4. Перезапустить Android Studio, при необходимости перезагрузить ПК.
5. Очистить кэш Gradle: **File** → **Invalidate Caches** → **Invalidate and Restart** (по желанию).
6. Снова собрать проект: **Build** → **Build APK(s)** или **Run**.

Когда все пути будут только латиница, ошибка `jlink.exe` и `JdkImageTransform` должна исчезнуть.
