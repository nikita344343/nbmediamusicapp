# Как установить приложение в LDPlayer 9

## Важно: APK в папке по умолчанию нет

Файла **app-debug.apk** в проекте изначально нет — он появляется только после первой сборки в Android Studio. Поэтому сначала нужно собрать проект.

---

## Шаг 1: Собрать APK в Android Studio

1. Открой папку **WordPressSyncApp** в Android Studio (**File** → **Open** → выбери эту папку).
2. Дождись синхронизации Gradle (внизу прогресс «Syncing…» / «Gradle sync finished»).
3. Меню **Build** → **Build Bundle(s) / APK(s)** → **Build APK(s)**.
4. Дождись окончания сборки (внизу: «APK(s) generated successfully»).
5. В уведомлении нажми **locate** — откроется папка с APK.  
   Либо открой вручную:  
   `WordPressSyncApp\app\build\outputs\apk\debug\app-debug.apk`  
   После первой сборки этот файл **появится** в этой папке.

## Шаг 2: Установить в LDPlayer 9

### Вариант А — перетаскивание (проще всего)

1. Запусти **LDPlayer 9**.
2. Перетащи файл **app-debug.apk** в окно эмулятора (на рабочий стол Android).
3. Подтверди установку, если спросит.
4. Иконка «WordPress Sync» появится в меню приложений.

### Вариант Б — через встроенный установщик LDPlayer

1. В правой панели LDPlayer нажми иконку **установки APK** (или меню с тремя полосками → «Установить APK»).
2. Укажи путь к файлу **app-debug.apk**.
3. Дождись установки.

### Вариант В — через ADB (если LDPlayer виден как устройство)

1. В настройках LDPlayer включи **отладку по USB** (Настройки → О телефоне → 7 раз «Номер сборки» → Разработчик → Отладка по USB).
2. В Android Studio при запущенном LDPlayer выбери его в списке устройств и нажми **Run** (зелёный треугольник) — приложение установится и запустится само.

---

Если после перетаскивания APK ничего не происходит — проверь, что в настройках LDPlayer разрешена установка из неизвестных источников (Настройки → Безопасность → Неизвестные источники / Установка из неизвестных источников).
